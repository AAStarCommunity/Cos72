# 社交恢复 / Social Recovery 说明

> 面向用户与团队的 guardian / 社交恢复机制说明。最后更新:2026-06-19。

## 一、社交恢复是什么

AirAccount 没有助记词。万一你换设备、丢了 passkey,**社交恢复**让你通过
**guardian(守护人)** 把账户的 signer 换成新设备,而不依赖任何一句助记词。

- 恢复采用 **2-of-3 门限**:配 3 个 guardian,任意 **2 个**批准即可恢复。
- **丢一个 guardian 不影响恢复**——只要还能凑够 2 个。这正是门限的意义。
- guardian 只是"能批准恢复的一方",它**借用一个能签名的身份**,本身不需要被保护(不递归,普通 AirAccount 即可当 guardian)。

## 二、guardian 的两个维度:信任假设 & 丢失风险

选什么当 guardian,就是在选 **你信任谁/什么** + **它可能怎么丢**:

| guardian 类型                 | 信任假设                   | 丢失风险                                                                                      |
| ----------------------------- | -------------------------- | --------------------------------------------------------------------------------------------- |
| **Apple passkey**             | 你的 iCloud 账户           | 低(跨 Apple 设备同步)                                                                         |
| **Google passkey**            | 你的 Google 账户           | 低(跨安卓设备同步)                                                                            |
| **MetaMask(自托管)**          | 只有你自己                 | **较高**——手机丢 / 助记词没记 = 没了                                                          |
| **朋友的 AirAccount**         | 那个朋友 + 其平台          | 低(朋友自己有备份)                                                                            |
| **passkey via KMS(当前实现)** | KMS 服务                   | 中(KMS 挂掉 → 该 guardian 签不了)。根治见 airaccount-contract#119(P-256 guardian 去 KMS 依赖) |
| **社区多签 guardian(默认)**   | 一群与你有真实社交关系的人 | **极低**(不绑定任何设备/密钥;见下)                                                            |

**原则:选多样化的 guardian**——不同方式、不同人、不同平台。这样任何单点失效(丢手机、某个平台、KMS)都无法一次性废掉所有 guardian。

## 三、社区多签 guardian(默认给每个人)

我们**默认给每个账户配一个"社区多签 guardian"**,作为 3 个 guardian 之一。于是典型组合是:

```
你的账户(2-of-3 社交恢复)
  ├─ 个人 guardian 1   (例:Apple passkey)
  ├─ 个人 guardian 2   (例:朋友的 AirAccount)
  └─ 社区多签 guardian (默认,社交验证)
```

**核心安全不变量(务必理解)**:社区 guardian
**只占 3 个里的 1 个,另外 2 个由你掌握**。恢复需要
**2 个**签名,所以**社区单独(1 个)永远够不到门限**——

> **即使社区作恶,也偷不走你的资产**(1 < 2)。

**为什么对你有用**:即使你**丢了一个个人 guardian**,仍可用"**剩下的那个个人 guardian + 社区多签 guardian**"凑够 2 个完成恢复。所以社区 guardian 是一张**只能帮你、不能害你**的社交安全网——不依赖任何设备/密钥。

### 社区多签 guardian 怎么验证"你是你"——靠社交关系,不靠密钥

社区 guardian 不是一把钥匙,而是一个**多签 + 社交验证**流程:

1. **它是一个多签**(例如 5 个社区成员)。
2. **每一次签名都是一场公开的会议**——过程公开、有记录、可追溯,不是私下操作。
3. **靠真实社交关系鉴别身份**:要让社区 guardian 为你的恢复签名,**与你有过真实、当面社交接触**的那些成员会**问你 3 个问题**。
   - 这 3 个问题**必须来自你们过往真实的社交场景**——真的发生过、当面经历过、只有真正的你才答得上来的事。
   - 这不是"密保问题"(那种能被猜/查),而是**多段真实社交关系**对"你是你"的共同确认。
4. **达到多签门限即生效**:比如 5 人多签里,有
   **3 个**与你有社交场景回忆、能为你作证的成员签了 → 社区多签 guardian 的批准就产生了。
5. 这个社区 guardian 批准 + 你**剩下的任意一个个人 guardian** = 2-of-3
   → 恢复完成。

### 这套设计的价值

- **身份锚定在多段真实社交关系上,而不是某个密钥/设备**——密钥会丢、会被偷、会被钓鱼;"一群当面认识你的人通过共同回忆认出你"不会。
- **公开会议 + 多签门限**:单个社区成员无法擅自冒充你发起恢复;需要多个与你有真实社交关系的人共同确认。
- **对普通人友好**:你不需要再保管一把"恢复钥匙";你的社交关系网就是你的恢复保险。

## 四、给用户的一句话

> 配
> **3 个 guardian、2 个批准就能恢复**;其中默认有一个**社区多签 guardian**(靠认识你的人当面用共同社交回忆确认你是你、公开多签)。所以**丢一个 guardian 也能恢复**。请给个人 guardian 选**多样化**的方式/人/平台,别都押在一个信任点上。
