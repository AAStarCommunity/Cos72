## Part 0 · 名字：为什么叫 "Cos72"（要简单可理解）

**Cos72 = cos(72°)** —— 黄金分割的角度（正五边形/黄金比例的核心角），象征**最佳协作模式**：把社区协作
调到最舒服、最可持续的那个"黄金比例"。同时谐音 **Co**operation **S**ystem（协作系统）—— 数学意象 + 语义双关。

- 首页用法："**把社区协作，调到黄金比例**" + 一行小字 "Cos72 取自 cos 72°，黄金分割的角度，象征最佳协作模式"。
- 不用"七十二变"（已弃）。

> **首页第一屏必须有一句"人话定义"**（注意：Cos72 **没有 Discord 那种聊天功能，有会员账号功能**）：
> **"Cos72 = 社区的共创系统：给你的社区配一套 会员账号 + 积分 + 兑换商城 + 声誉体系，开源、免费、数据和积分都归社区自己。"**

## Part 1 · 三问定稿（在你草稿上加"类比熟悉概念"）

### ① 为谁提供什么服务
**对象**：兴趣、社交、话题小组等**社区类组织**。两种角色、不同权限：
| 角色 | 做什么 | 类比熟悉的东西 |
|---|---|---|
| **社区管理员** | 办活动/运营、成员 onboarding、发激励、派任务并检查 | "社区版的 **会员运营后台 + 积分/任务系统**" |
| **社区成员** | 了解社区、参与并记录活动、做任务拿积分、积分兑咖啡/Swag/奖品 | "**会员账号 + 集章打卡 + 积分商城**" |

> 注：Cos72 **不做 Discord 那种聊天**，核心是**会员账号 + 贡献/积分/声誉**。

### ② 为何用 Cos72（独有价值）
**低摩擦、开源**的系统，用两样东西激活参与和贡献、沉淀连接、提升社区可持续：
| 机制 | 是什么 | 类比 |
|---|---|---|
| **Reputation / NFT** | 社区内 + **跨社区**通用的声誉 | "**跨平台通用的荣誉徽章 / 信用分**"（不像微信群，换个群就归零） |
| **xPNTs** | 社区内（不可跨社区）的贡献记录 | "**游戏公会的贡献值**"，只在本社区有效、链上可信、社区自主发行 |

一句话："**参与留痕、贡献可信、荣誉带得走** —— 让社区的连接沉淀下来，而不是活动一散就清零。"

### ③ 如何用（分层）
首页轮播（小J 叙事）→ tour（系统概述 + 角色手把手）→ 角色流程图 → FAQ → 可搜索手册。见 Part 2。

## Part 3 · 首页设计 —— 几套方案（选一或混合）

**方案 A：三屏轮播（最快建立认知）**
- 3 张轮播 = 三问，每张：小J 一个姿势 + 一句关键词 + 一张示意图 + 一行类比。
- 优点：极简、移动端友好、10 秒扫完。缺点：叙事偏浅。

**方案 B：单页滚动叙事（仿 tour.mushroom.cv）**
- Hero（人话定义 + 小J）→ 往下滚三段（谁/为何/如何），每段一图一句 + 类比卡片。
- 优点：故事性强、和 tour 风格统一。缺点：要滚动、内容更多。

**方案 C：渐进式（访客看 landing，回访跳 dashboard）**
- 未登录 → 落地页（A 或 B）；已登录 → 直接进 dashboard（保留当前逻辑）。
- 优点：老用户零打扰、新用户有认知。**建议所有方案都叠加 C 这层判断**。

**推荐**：**C（渐进） + A（轮播）打底**，Hero 一句人话定义 + 3 轮播卡；想要更强叙事再升级到 B。三问的示意图可复用 tour 的漫画 panel。

## Part 4 · Tour 挪移计划（Astro 静态 → YAAA 里的静态页）

**现状**：`~/Dev/aastar/cos72-tour` 是 **Astro** 站，`pnpm build → dist/`（纯静态 HTML + `/panels` 漫画图）。角色数据 `src/data/roles/*.ts`（visitor/individual/community-admin/operator/guardian/protocol-admin）。

**三种挪法：**
| 方案 | 做法 | 优点 | 缺点 |
|---|---|---|---|
| **① 构建产物直挂（最快）** | `astro build` → 把 `dist/` 拷进 YAAA `public/tour/` → cos72.aastar.io/tour 直出静态 | 改动最小、样式自带、几小时能上 | 两套源码；更新 tour 要重新拷 |
| **② 源码搬入 + 子构建** | tour 源码放 YAAA 子目录，CI 里 astro build 到 public/tour | 单仓、可版本化 | 引入 Astro 工具链到 YAAA（pnpm/astro）|
| **③ 重写为 Next 页面** | 把 roles 数据 + panels 移植成 Next 的 `/tour` 路由 | 技术栈统一、和 YAAA 深度整合（登录态、i18n）| 工作量最大、重写叙事组件 |

**建议**：**先①快速上线**（拷 dist 到 public/tour，验证内容与链接），**再逐步③**把最需要和登录态/i18n 整合的角色流程重写为 Next 页面。L4/L5（FAQ/手册）用 Next 页 + 可搜索（本地 JSON 索引，符合 [[frontend-client-side-static-preference]] 客户端静态）。

**"用户手册"落点**（你说的 tour 首页加 user manual + 每角色/功能列表）：在 `/tour` 首页加一个 **User Manual 区** → 列出"角色 × 功能"表（功能名 + 一句解释 + "查看流程"链接到 L3 独立页，图=系统截图）。截图需要跑一遍真实系统抓图（可等你 onboarding 验证通过后用 cos72 抓）。
